Image upload and filtering

This is the login page for a site that uses Python/Flask-rendered (using Jinja templating) page to take an uploaded image from the user, and apply filters to it.

User accounts are stored in a backend SQLite3 database. If the user account does not exist, users cannot login. Instead, they must create one. Users need to enter their password twice. JavaScript validates that they have entered the same password each time.

After logging in, users gain access to the image upload and filter functions (See part two). The Python backend is written to prevent access to any of the static pages without logging in. E.g. a user cannot simply navigate to "uploaded.html".
